# 情感分析工作台

## 工作台定位
应用层。负责判断一段文本表达的是积极、消极还是中性的态度。

这页是应用层里非常重要的一站，因为它看起来像“更具体的文本分类”，但实际上会暴露很多自然语言处理中最典型的难点：
- 否定词
- 反讽
- 语境
- 多义词

所以情感分析既适合作为应用案例，也适合拿来讲边界。

## 一、什么是情感分析？
情感分析就是：

判断一段文本表达的是正面态度、负面态度，还是中性态度。

例如：
- “这门课非常有帮助。” -> 积极
- “设备老坏，体验很差。” -> 消极
- “这门课每周上两次。” -> 中性

## 二、为什么它常被看作文本分类的重要应用？
因为它本质上就是一个分类任务。

只是这里的类别不再是：
- 体育
- 财经
- 科技

而是：
- positive
- negative
- neutral

所以可以把情感分析理解成：
- 文本分类的一个典型落地场景

## 三、典型场景有哪些？
### 1. 商品评论
判断用户对商品是喜欢还是不喜欢。

### 2. 课程反馈
判断学生评价整体偏正面还是负面。

### 3. 电影评论
判断观众喜欢还是不喜欢一部电影。

### 4. 社交媒体
分析公众对某个人、某个事件、某个品牌的情绪倾向。

## 四、什么是情感极性？
情感极性就是：

把情绪方向转成一个更可计算的概念。

通常分成三类：
- 积极 positive
- 消极 negative
- 中性 neutral

有些工具还会给一个分数，例如：
- 接近 1 表示更积极
- 接近 -1 表示更消极
- 接近 0 表示更中性

## 五、词典法和监督学习法有什么区别？
这是这页最核心的知识点之一。

### 1. 词典法
词典法的思路是：
- 先准备一份情感词典
- 词典里记录哪些词偏正面，哪些词偏负面
- 计算文本中这些词带来的总体倾向

例如：
- `good` 偏正面
- `bad` 偏负面
- `wonderful` 偏正面

#### 词典法适合什么场景？
- 没有标注数据时
- 想快速做一个入门实验时
- 想先快速体验情感分析时

#### 词典法的局限
- 只看词，不一定理解上下文
- 容易被反讽误导
- 对否定词不够敏感时容易出错
- 对领域语言不一定适应

### 2. 监督学习法
监督学习法的思路是：
- 先准备很多带标签的文本
- 让模型从数据中自己学习规律

例如：
- 这条评论标签是 positive
- 那条评论标签是 negative

模型通过训练后，再去预测新文本的情感类别。

#### 监督学习法适合什么场景？
- 手头有标注数据
- 想追求更高准确率
- 想做更稳定的应用系统

#### 监督学习法的局限
- 需要标注数据
- 训练过程更复杂
- 数据质量不高时效果会差

## 六、为什么情感分析比看起来更难？
### 1. 否定词会反转意思
例如：

```text
This course is good.
This course is not good.
```

只多了一个 `not`，情绪方向就变了。

所以在情感分析里，不能像其他任务那样轻易删掉否定词。

### 2. 反讽会误导系统
例如：

```text
Great, another broken projector. Wonderful.
```

表面上有：
- great
- wonderful

但其实是差评。

这对词典法尤其困难。

### 3. 语境会改变词义
同一个词在不同句子里，情绪可能不同。

例如：
- `ridiculous` 有时是负面
- 但在口语里也可能表示“夸张得厉害”，语气未必完全负面

### 4. 多义词会带来歧义
词本身不是固定带情绪的，得看它和谁一起出现。

## 七、一个适合课堂的词典法例子
对初学者来说，可以先用 `TextBlob` 快速体验。

```python
from textblob import TextBlob

review1 = "This movie is absolutely wonderful and I loved it!"
review2 = "The food was terrible and the service was bad."

score1 = TextBlob(review1).sentiment.polarity
score2 = TextBlob(review2).sentiment.polarity

print("评论1得分：", score1)
print("评论2得分：", score2)
```

这段代码适合课堂第一次演示，因为它能马上看到：
- 正面文本分数偏正
- 负面文本分数偏负

## 八、一个最简单的监督学习思路
如果你前面已经学过文本分类，就可以把情感分析看成一个二分类任务：

```python
texts = [
    "I love this course",
    "This class is boring",
    "The teacher is helpful",
    "The classroom is terrible"
]

labels = [1, 0, 1, 0]
```

然后继续走：
- 清洗
- 向量化
- 训练分类器
- 评估结果

也就是说，情感分析和文本分类的流程高度相似。

## 九、适合课堂的对比例子：看反讽怎么误判

```python
from textblob import TextBlob

text = "Great, the system crashed again. Wonderful."
score = TextBlob(text).sentiment.polarity
print(score)
```

这个例子很适合课堂讨论，因为它能让学生意识到：
- 词典法并不真正“理解”语气

## 十一、初学者最常见错误
### 1. 清洗时误删否定词
例如把：
- `not`
- `no`

当成普通停用词删掉。

这会直接破坏情感方向。

### 2. 乱删标点和表情
在情感分析里：
- `!`
- `?`
- 表情符号

有时携带很强的情绪线索。

### 3. 只看总分，不看文本内容
模型给出的分数只是结果，不等于解释。

### 4. 以为词典法能处理所有复杂语气
其实它对：
- 反讽
- 语境变化
- 多义词

都比较脆弱。

### 5. 把情感分析当成完全客观的判断
很多文本本来就有模糊性，不同人也可能给出不同标签。
