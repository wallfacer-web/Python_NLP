# 主题分析与摘要工作台

## 工作台定位
应用层。负责回答两类非常常见的问题：
- 这批文本主要在说什么
- 这篇长文本能不能压缩成更短的版本

前者偏向主题分析，后者偏向摘要。

这两个任务经常一起出现，但它们不是一回事。

## 一、什么是主题分析？
主题分析的目标是：

从一批文本里发现“经常一起出现”的词和潜在主题。

例如你有很多课程反馈，系统可能发现一组经常一起出现的词：
- 作业
- 难度
- 截止时间
- 压力

这就提示你：
- 这批反馈里可能有一个“作业负担”主题

## 二、什么是摘要？
摘要的目标是：

把一篇长文本压缩成更短、更容易阅读的版本，同时尽量保留核心信息。

例如：
- 把一篇长文章压缩成几句话
- 把一段讲座记录压缩成一个段落
- 把一份访谈压缩成要点

## 三、主题分析和摘要有什么区别？
最简单的理解方式是：

- 主题分析：看“一批文本主要谈哪些方面”
- 摘要：看“一篇长文本的核心内容是什么”

所以：
- 主题分析通常面对的是文本集合
- 摘要通常面对的是单篇长文本或少量文本

## 四、两个关键边界
### 1. 主题不等于标题
主题分析输出的往往不是一句现成的人类语言标题。

它更常见的结果是：
- 一组高频共现词

例如：

```text
['作业', '截止时间', '压力', '反馈']
```

这时候真正的人类分析工作是：
- 看到这组词后
- 给它解释成“作业负担”或“课程节奏压力”

所以主题是“机器发现的词组结构”，不等于“现成标题”。

### 2. 摘要不等于简单截句
真正的摘要不是：
- 只把前两句复制下来
- 只截取一小段原文

摘要的核心是：
- 抓住重要内容
- 压缩信息
- 尽量保留整体意义

## 五、主题分析适合什么场景？
主题分析特别适合：
- 大量评论
- 新闻集合
- 课程反馈
- 访谈文本集合
- 问卷开放题答案

因为这类任务往往不是想看一篇，而是想从很多文本里找出“主要问题”和“主要方向”。

## 六、摘要适合什么场景？
摘要特别适合：
- 长文章
- 讲座记录
- 访谈内容
- 报告
- 论文长段落

因为这类任务的核心痛点是：
- 太长，读不过来

## 七、主题分析可以怎样通俗理解？
可以把主题分析理解成：

在一大堆文本里，找出哪些词经常结伴出现。

如果一组词经常一起出现，它们背后可能对应一个隐藏主题。

例如：
- `篮球、足球、比赛、球队`

可能对应“体育”主题。

再例如：
- `屏幕、电池、充电、续航`

可能对应“电子产品性能”主题。

## 八、抽取式摘要和生成式摘要有什么区别？
### 1. 抽取式摘要
抽取式摘要的思路是：
- 从原文里挑重要句子
- 直接拼成摘要

优点：
- 不容易捏造原文没有的信息
- 比较稳定

缺点：
- 句子拼接后可能不自然
- 连贯性有时不够好

### 2. 生成式摘要
生成式摘要的思路是：
- 先理解文本大意
- 再用新的话重新表达

优点：
- 更自然
- 更像人写的摘要

缺点：
- 可能出现“幻觉”
- 有时会加入原文没有的内容

所以：
- 抽取式更稳
- 生成式更流畅

## 九、一个最小关键词/主题观察例子
关键词提取可以作为主题分析的最简入口。

```python
from gensim.summarization import keywords

text = """
This course includes Python programming, natural language processing,
text cleaning, text classification, and sentiment analysis.
"""

print(keywords(text))
```

这个例子适合入门，因为它能让学生先感受到：
- 机器可以从文本里抓出更代表核心内容的词

## 十、一个最小摘要例子
用 `gensim` 的抽取式摘要可以快速演示：

```python
from gensim.summarization import summarize

long_text = """
The game's main story revolves around the player character's quest to defeat
Alduin the World-Eater, a dragon who is prophesied to destroy the world.
Over the course of the game, the player completes quests and develops
their character.
"""

summary = summarize(long_text, ratio=0.5)
print(summary)
```

这个例子适合讲：
- 摘要是从原文中挑重要句子
- 不是重新生成

## 十一、一个最小主题分析例子
用 `scikit-learn` 做一个非常小的 LDA 演示：

```python
from sklearn.decomposition import LatentDirichletAllocation
from sklearn.feature_extraction.text import CountVectorizer

corpus = [
    "I love apples and bananas",
    "Strawberries and grapes are delicious fruits",
    "Football and basketball are exciting sports",
    "Basketball is my favorite sport"
]

vectorizer = CountVectorizer()
X = vectorizer.fit_transform(corpus)

lda = LatentDirichletAllocation(n_components=2, random_state=42)
lda.fit(X)

print("主题模型训练完成")
```

这段代码最适合课堂的作用不是“马上看懂 LDA 数学”，而是先让学生知道：
- 一堆文本可以被自动归到几个潜在主题方向


## 十三、初学者最常见错误
### 1. 对单篇短文做主题分析
主题分析更适合一批文本，而不是一篇很短的文章。

### 2. 不清理停用词
如果不做基础清洗和停用词处理，主题结果里会出现很多废词。

### 3. 以为摘要一定流畅自然
抽取式摘要常常会有“拼接感”。

### 4. 把主题词直接当最终结论
主题词只是中间结果，分析者还要负责解释。

### 5. 只看结果，不回看原文
无论是主题分析还是摘要，都不能脱离原文人工复查。
