# 文本分类工作台

## 工作台定位
应用层。负责把一段文本自动分到预先设定好的类别里。

前面的工作台主要在解决：
- 文本怎么读进来
- 文本怎么清洗
- 文本里能抽什么
- 文本里哪些词高频

这一页开始进入一个非常典型的 NLP 应用任务：

让计算机根据已有样本，学会给新文本贴标签。

## 一、什么是文本分类？
文本分类就是：

把一段文本放进一个预先定义好的类别里。

例如：
- 这条新闻属于“体育”还是“娱乐”
- 这封邮件是“正常邮件”还是“垃圾邮件”
- 这条课程评价属于“教学问题”还是“设施问题”

所以它本质上是一个“自动分门别类”的任务。

## 二、为什么它是最常见的 NLP 任务之一？
因为现实世界里，大量文本都需要先被分类，后面才能处理。

常见场景包括：
- 新闻自动分栏
- 邮件过滤
- 客服工单分类
- 课程反馈归类
- 评论情感判断

它既实用，又能把前面学过的很多基础动作串起来，所以非常适合放在应用层开头。

## 三、典型场景有哪些？
### 1. 垃圾邮件识别
判断一封邮件是：
- 正常
- 垃圾

### 2. 新闻分类
判断一篇新闻属于：
- 体育
- 财经
- 科技
- 娱乐

### 3. 课程评价分类
判断一条反馈属于：
- 教学内容
- 作业安排
- 设备问题
- 服务问题

### 4. 情感分类
判断一段评论是：
- 积极
- 消极
- 中立

这里要顺手强调：

情感分析其实就是文本分类的一个典型子任务。

## 四、什么是标签数据？
标签数据就是：

文本旁边已经有人类提前写好了“标准答案”。

例如：

| text | label |
|---|---|
| This course is very helpful. | positive |
| The classroom equipment is broken. | facility |

这里：
- `text` 是文本
- `label` 是标签

如果没有标签，监督学习式的文本分类就没法直接开始。

## 五、什么是训练集和测试集？
### 1. 训练集
可以理解成：
- 机器学习时用的“练习题”

模型通过训练集去学习：
- 哪些词和哪个标签更相关

### 2. 测试集
可以理解成：
- 学完后拿来考试的新题

测试集不能在训练时提前偷看。

这是非常重要的一条规则。

如果提前看过，模型成绩会虚高，结果不可信。

## 六、最基础的文本分类流程
一个最基础的文本分类流程通常是：

1. 准备文本和标签  
2. 清洗文本  
3. 把文本变成数字向量  
4. 训练分类器  
5. 评估结果  

这五步比具体算法更重要，因为它构成了完整工作流。

## 七、为什么文本要先变成数字？
因为机器学习模型不会直接理解自然语言字符串。

它更擅长处理：
- 数字
- 向量
- 表格矩阵

所以文本分类前，必须先把文本转换成数值特征。

## 八、Bag of Words 是什么？
`Bag of Words` 可以理解成：

只看“这篇文本里哪些词出现了、出现了多少次”，不管词序。

例如：

```text
python is useful
```

可能会被表示成某种词频向量。

它像一个装词的袋子：
- 词在不在
- 出现几次

但不关心句法顺序。

这也是它的优点和局限：
- 优点：简单直观
- 局限：忽略词序和深层语义

## 九、TF-IDF 是什么？
`TF-IDF` 也是把文本变成数字的一种方法。

和普通词频不同，它会降低那些“到处都常见”的词的权重，提高那些“在某篇文本里更有区分度”的词的权重。

可以把它理解成：
- 词频：这篇文章里提了几次
- 逆文档频率：这个词是不是全世界都在提

所以它通常比纯词频更适合分类任务。

## 十、一个最基础的分类代码示例
下面这段代码适合做课堂第一次演示：

```python
from sklearn.model_selection import train_test_split
from sklearn.feature_extraction.text import CountVectorizer
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score

texts = [
    "This food is amazing and delicious",
    "Terrible experience, worst food ever",
    "I love this place, highly recommended",
    "Awful service, never going back"
]

labels = [1, 0, 1, 0]

X_train, X_test, y_train, y_test = train_test_split(
    texts, labels, test_size=0.5, random_state=42
)

vectorizer = CountVectorizer()
X_train_vec = vectorizer.fit_transform(X_train)
X_test_vec = vectorizer.transform(X_test)

model = LogisticRegression()
model.fit(X_train_vec, y_train)

pred = model.predict(X_test_vec)

print("预测结果：", pred)
print("真实标签：", y_test)
print("准确率：", accuracy_score(y_test, pred))
```

## 十一、一步一步解释这段代码
### 1. 准备文本和标签
`texts` 是文本列表，`labels` 是对应标签。

### 2. 划分训练集和测试集

```python
train_test_split(...)
```

意思是：
- 一部分拿来学习
- 一部分留着测试

### 3. 向量化

```python
CountVectorizer()
```

意思是：
- 把文本变成词袋特征

### 4. 训练模型

```python
model.fit(...)
```

意思是：
- 用训练集学习分类规律

### 5. 预测

```python
model.predict(...)
```

意思是：
- 给测试集预测标签

### 6. 评估

```python
accuracy_score(...)
```

意思是：
- 看模型做对了多少

## 十二、如何换成 TF-IDF
只要把：

```python
from sklearn.feature_extraction.text import CountVectorizer
```

换成：

```python
from sklearn.feature_extraction.text import TfidfVectorizer
```

再把：

```python
vectorizer = CountVectorizer()
```

换成：

```python
vectorizer = TfidfVectorizer()
```

就可以了。

这很适合做课堂对比实验。

## 十三、准确率、精确率、召回率、F1 直观是什么意思？
### 1. 准确率 Accuracy
所有题里，一共做对了多少。

### 2. 精确率 Precision
模型说“这是正类”的那些文本里，真正是正类的比例。

可以理解成：
- 不要乱抓

### 3. 召回率 Recall
所有真正的正类里，模型抓出来了多少。

可以理解成：
- 不要漏抓

### 4. F1
精确率和召回率的综合平衡分数。

当类别不平衡时，F1 比单看准确率更有参考价值。

## 十四、一个更完整的评估示例

```python
from sklearn.metrics import classification_report

print(classification_report(y_test, pred))
```

这个输出里通常会包含：
- precision
- recall
- f1-score

课堂上不需要一下讲很深，但至少要让学生知道：
- 不能只看 accuracy

## 十六、初学者最常见错误
### 1. 数据泄露
最典型错误是：
- 在切分训练集和测试集之前，就对全部数据做了 `fit`

这相当于模型提前偷看了考试内容。

### 2. 测试集错误使用 `fit_transform()`
正确原则：
- 训练集：`fit_transform()`
- 测试集：`transform()`

### 3. 文本清洗不充分
如果文本里还有大量噪声，向量化后的特征就会很脏。

### 4. 只看准确率
特别是在类别不平衡时，准确率可能会误导人。

### 5. 数据太少却急着下结论
小样本只能做演示，不代表模型真的稳定。
