# ASR与TTS工作台

## 工作台定位
应用层。负责把语音和文本互相转换。

这一页包含两条非常重要的应用链：
- `ASR`：语音转文本
- `TTS`：文本转语音

如果前面的课程主要在处理“已经是文本的材料”，那么这一页开始让学生接触：
- 声音如何进入 NLP 流程
- 文本如何重新变成语音输出

## 一、什么是 ASR？什么是 TTS？
### 1. ASR
`ASR` 是 `Automatic Speech Recognition`，中文通常叫：
- 自动语音识别

它的作用是：
- 把人说的话变成文字

可以把它理解成：
- 计算机的耳朵
- 计算机的速记员

### 2. TTS
`TTS` 是 `Text-to-Speech`，中文通常叫：
- 文本转语音

它的作用是：
- 把文字念出来

可以把它理解成：
- 计算机的嘴巴
- 计算机的播音员

### 3. 两者的区别
- `ASR`：声音 -> 文本
- `TTS`：文本 -> 声音

## 二、典型应用场景有哪些？
### ASR 常见场景
- 课程录音转写
- 访谈转写
- 自动字幕生成
- 会议纪要整理
- 语音输入

### TTS 常见场景
- 文本朗读
- 无障碍阅读
- 语音助手
- 自动播报
- 学习材料朗读

## 三、为什么 ASR 对 NLP 很重要？
因为很多真实语言材料一开始并不是文本，而是声音。

例如：
- 教师讲课录音
- 访谈音频
- 视频字幕原始音轨
- 口语语料

ASR 的作用就是先把这些声音变成文本，后面才能接上：
- 文本清洗
- 情感分析
- 主题分析
- 摘要
- 大模型处理

所以它其实是：
- 语音进入文本工作流的入口

## 四、为什么音频格式、采样率、噪声、口音、停顿会影响 ASR 效果？
### 1. 音频格式
不同工具支持的格式不同，例如：
- `.wav`
- `.mp3`
- `.m4a`

如果格式不兼容，程序可能直接读不了。

### 2. 采样率
采样率可以简单理解成：
- 声音被记录得有多细

如果采样率太低，语音细节可能丢失。

### 3. 噪声
如果背景里有：
- 风声
- 电流声
- 教室杂音
- 说话重叠

识别效果会明显变差。

### 4. 口音
ASR 往往在标准发音上表现更稳定。

如果说话人带有较强口音，识别错误会更多。

### 5. 停顿和语速
说话太快、停顿混乱、句子边界不清楚，都会增加转写难度。

## 五、为什么 ASR 输出的文本还要进一步清洗？
因为 ASR 转出来的文本通常不是“最终干净文本”。

常见问题包括：
- 标点不稳定
- 分句不理想
- 专有名词识别错误
- 同音词误识别
- 口头语很多
- 重复词或停顿词较多

所以 ASR 输出之后，通常还要继续做：
- 文本清洗
- 关键词提取
- 情感分析
- 摘要

## 六、TTS 的大致思路是什么？
不讲复杂技术，可以先这样理解：

TTS 的任务是：
- 先读懂文本里写了什么
- 再根据发音规则和语音模型，把它合成为可听的音频

它不是简单把每个字母拼起来，而是要考虑：
- 发音
- 停顿
- 节奏
- 语调

所以好的 TTS 会听起来更自然。

## 七、课程推荐工具：OpenAI Whisper 与微软 edge-tts
这门课里，ASR 和 TTS 我建议优先使用下面两套工具：

- `OpenAI Whisper`：做 ASR

原因很简单：
- 安装相对直接
- 例子多
- Python 里容易调用
- 很适合教学演示

## 八、Whisper 的基本原理
根据 OpenAI 的官方仓库，Whisper 是一个通用语音识别模型，训练于大量多样化音频数据，能够完成：
- 多语言语音识别
- 语音翻译
- 语言识别

它本质上是一个基于 Transformer 的序列到序列模型。

对教学来说，可以先把它理解成：
- 模型先“听”音频
- 再一步步预测应该输出哪些文本 token

Whisper 的一个重要特点是：
- 它不是只为一种语言设计
- 它对多语言、真实环境语音相对更稳

## 九、Whisper 的安装
### 1. 安装 Python 包

```bash
pip install -U openai-whisper
```

### 2. 安装 ffmpeg
Whisper 官方 README 明确要求安装 `ffmpeg`。

Windows 常见方式：


或者：


### 3. 如果环境里还没有 PyTorch
可以先安装适合自己环境的 `torch`。

教学里如果只做入门演示，先确认：
- Python 可用
- `openai-whisper` 安装成功
- `ffmpeg` 在 PATH 中可用

## 十、Whisper 的使用例子
### 1. 命令行例子

```bash
whisper demo.wav --model turbo
```

如果是中文音频，也可以显式指定语言：

```bash
whisper demo.wav --language Chinese --model small
```

### 2. Python 例子

```python
import whisper

model = whisper.load_model("small")
result = model.transcribe("demo.wav", language="zh")

print(result["text"])
```

### 3. 代码解释
- `load_model("small")`：加载一个较适合教学演示的模型
- `transcribe(...)`：把音频转成文本
- `result["text"]`：取出转写结果

### 4. 课堂提醒
- 英文快速演示可以用 `turbo`
- 中文或多语言更适合先用 `small`、`medium` 这类多语言模型做稳定演示
- 音频越干净，效果通常越好

## 十一、edge-tts 的基本原理
根据 `edge-tts` 官方仓库，它是一个 Python 模块，可以直接调用 Microsoft Edge 的在线 TTS 服务，而且：
- 不需要安装 Edge 浏览器
- 不需要 Windows
- 不需要单独申请 API key

对教学来说，可以把它理解成：
- 你把文本和语音角色发给服务
- 服务返回一段合成好的语音

所以它本质上更像：
- 一个“在线语音合成客户端”

## 十二、edge-tts 的安装

```bash
pip install edge-tts
```

如果只想用命令行，也可以用：

```bash
pipx install edge-tts
```

## 十三、edge-tts 的使用例子
### 1. 先查看可用声音

```bash
edge-tts --list-voices
```

### 2. 命令行合成一个 mp3

```bash
edge-tts --voice zh-CN-XiaoxiaoNeural --text "你好，欢迎来到 Python 与自然语言处理课程。" --write-media hello.mp3
```

### 3. 调整语速、音量、音高

```bash
edge-tts --voice zh-CN-XiaoxiaoNeural --rate=-20% --text "今天我们学习语音转文本和文本转语音。" --write-media demo.mp3
```

### 4. Python 例子

```python
import asyncio
import edge_tts

TEXT = "你好，欢迎来到 Python 与自然语言处理课程。"
VOICE = "zh-CN-XiaoxiaoNeural"

async def main():
    communicate = edge_tts.Communicate(TEXT, VOICE)
    await communicate.save("hello.mp3")

asyncio.run(main())
```

### 5. 代码解释
- `Communicate(TEXT, VOICE)`：创建一次语音合成任务
- `save("hello.mp3")`：把结果保存成 mp3

## 十四、Whisper 与 edge-tts 的组合演示
这两个工具很适合一起做课堂小项目：

1. 学生先录一小段语音  
2. 用 `Whisper` 把语音转成文本

这会让学生非常直观地看到：
- 声音 -> 文本 -> 分析 -> 声音

## 十六、初学者最常见错误
### 1. Whisper 安装了，但没装 ffmpeg
这会导致音频处理直接失败。

### 2. 音频格式不合适
格式不兼容，或者文件损坏，程序就没法正常读入。

### 3. 误以为 Whisper 输出一定完全正确
背景噪声、口音、专有名词都会带来识别错误。

### 4. edge-tts 声音名写错
如果 voice 名不对，命令就无法正常生成语音。

### 5. 把 ASR 和 TTS 当成最终结果，而不是工作流的一部分
它们更适合被看作：
- 输入层
- 输出层

中间仍然需要文本处理。

## 十七、它和前后工作台是什么关系？
### 和前面的关系
前面的 `文本清洗工作台` 会在 ASR 之后继续发挥作用，因为转写文本通常需要清理。

前面的 `情感分析工作台`、`主题分析与摘要工作台` 也可以直接接在 ASR 输出之后。

也就是说：
- 语音先变文本
- 文本再进入前面已经搭好的 NLP 流程

### 和后面的关系
后面的 `大模型文本处理工作台` 可以进一步做：
- 会议内容摘要
- 讲座问答
- 访谈要点提取

而 `RAG工作台` 可以把转写后的大量语音资料变成可检索知识库。
