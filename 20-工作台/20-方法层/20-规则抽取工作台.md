# 规则抽取工作台

## 工作台定位
方法层。负责使用规则和正则表达式，从文本中提取结构比较稳定的信息。

这一个工作台非常适合初学者，因为它能让学生很快看到“文本变结构化结果”的过程。

它不是在猜意思，而是在找模式。

- 方法层

## 它对应什么输出动作？
- 抽取邮箱、日期、网址、手机号、课程代码等结构化字段
- 提取关键词或简单模式
- 输出抽取结果表
- 记录规则说明和错误案例

## 一、什么是规则抽取？
规则抽取就是：

人先观察文本里的规律，再把规律写成规则，让计算机按规则把目标信息抓出来。

例如：
- 邮箱里通常有 `@`
- 日期常有固定格式
- 课程代码可能是“大写字母 + 数字”
- 网址常以 `http://` 或 `https://` 开头

这类任务很适合规则法，因为目标的“外形”比较稳定。

## 二、为什么规则抽取在 NLP 入门阶段很重要？
### 1. 见效快
不需要训练模型，不需要标注大量数据，只要规则写对，马上就能抽到结果。

### 2. 容易理解
学生可以清楚看到：
- 原始文本是什么
- 规则写了什么
- 程序为什么抓到了这些结果

### 3. 是后续很多方法的基础
后面的机器学习、spaCy、信息抽取工作流，并不是完全替代规则。

在真实工作里，规则常常用来：
- 清洗数据
- 做初步抽取
- 统一输出格式
- 给机器学习结果做后处理

## 三、什么情况下适合用规则法？
规则法最适合：
- 目标形式稳定
- 文本结构比较规律
- 任务要求清楚

典型例子：
- 数字
- 日期
- 邮箱
- 电话号码
- 网址
- 课程代码
- 明确的关键词

例如：
- `alice@example.com`
- `2026-03-23`
- `ENG101`

这些对象都有比较明显的外形。

## 四、什么情况下不适合只用规则法？
当文本的意义太依赖上下文，或者表达方式变化太大时，规则法就不够用了。

例如：
- 判断一句话是不是讽刺
- 找出一篇新闻的主要人物
- 判断一句评价是真夸还是反话
- 提取复杂关系

这类任务往往需要：
- 机器学习
- 统计方法
- 词性和实体识别
- 语义理解

所以规则法不是万能的，但它非常适合做入门和基础工程。

## 五、正则表达式在规则抽取里是什么？
正则表达式可以理解成一张“模式说明书”。

普通查找像这样：

```python
"python" in text
```

它只能找一个具体词。

正则表达式更像是：

“找所有满足某种外形条件的内容。”

比如：
- 所有连续数字
- 所有像邮箱的字符串
- 所有像日期的片段

所以可以把正则表达式想成：
- 一个高级筛子
- 一个模式化的查找工具

## 六、最基础的抽取对象
### 1. 数字

```python
r"\d+"
```

表示：
- 一个或多个数字

### 2. 日期

```python
r"\d{4}-\d{2}-\d{2}"
```

例如匹配：
- `2026-03-23`

### 3. 邮箱

初学阶段可以先用一个简化版：

```python
r"[a-zA-Z0-9._%+-]+@[a-zA-Z0-9.-]+\.[a-zA-Z]+"
```

### 4. 网址

```python
r"https?://\S+"
```

### 5. 手机号或电话

例如固定格式电话：

```python
r"\d{3}-\d{3}-\d{4}"
```

### 6. 课程代码

例如：
- `ENG101`
- `NLP202`

可以写成：

```python
r"[A-Z]{3}\d{3}"
```

### 7. 关键词
如果想同时匹配多个关键词，可以用 `|` 表示“或”：

```python
r"Python|Java|C\+\+"
```

## 七、Python 的 re 模块最常用的三个方法
先导入模块：

```python
import re
```

### 1. `re.search()`
作用：
- 只找第一个匹配

```python
import re

text = "课程代码是 ENG101。"
match = re.search(r"[A-Z]{3}\d{3}", text)

if match:
    print(match.group())
```

注意：
- `search()` 返回的是匹配对象
- 真正的文本要用 `.group()` 取出来

### 2. `re.findall()`
作用：
- 找出所有匹配内容
- 返回列表

```python
import re

text = "价格有 299 元、99 元、15 元。"
numbers = re.findall(r"\d+", text)
print(numbers)
```

这是规则抽取里最常用的方法。

### 3. `re.sub()`
作用：
- 查找并替换

```python
import re

text = "请联系 415-555-1234。"
masked = re.sub(r"\d{3}-\d{3}-\d{4}", "XXX-XXX-XXXX", text)
print(masked)
```

这很适合：
- 隐藏隐私信息
- 删除噪声
- 统一格式

## 八、适合课堂的最小例子
### 例子1：提取所有数字

```python
import re

text = "这件衣服原价 299 元，现在只要 99 元，邮费 15 元。"
result = re.findall(r"\d+", text)
print(result)
```

### 例子2：提取邮箱

```python
import re

text = "联系邮箱：alice@example.com 和 bob_123@work.org"
emails = re.findall(r"[a-zA-Z0-9._%+-]+@[a-zA-Z0-9.-]+\.[a-zA-Z]+", text)
print(emails)
```

### 例子3：提取网址

```python
import re

text = "官网是 https://example.com，文档页是 http://docs.test.org/page"
urls = re.findall(r"https?://\S+", text)
print(urls)
```

### 例子4：提取课程代码

```python
import re

text = "本学期开设 ENG101、NLP202 和 HIS305。"
codes = re.findall(r"[A-Z]{3}\d{3}", text)
print(codes)
```

### 例子5：隐藏电话号码

```python
import re

text = "有事请拨打 415-555-1234。"
new_text = re.sub(r"\d{3}-\d{3}-\d{4}", "XXX-XXX-XXXX", text)
print(new_text)
```

### 例子6：先清洗，再抽取

```python
import re

text = "  Contact me at Alice@example.com \n"
text = text.strip().lower()
emails = re.findall(r"[a-zA-Z0-9._%+-]+@[a-zA-Z0-9.-]+\.[a-zA-Z]+", text)
print(emails)
```

这个例子很重要，因为它把前面的“文本清洗工作台”和当前工作台连起来了。

## 十、初学者最常见错误
### 1. 忘记 `import re`

### 2. 忘记使用原始字符串 `r""`
推荐这样写：

```python
r"\d+"
```

而不是：

```python
"\\d+"
```

前者更清楚，也更不容易出错。

### 3. 混淆 `search()` 和 `findall()`
- `search()` 找第一个
- `findall()` 找全部

### 4. `search()` 后忘记 `.group()`

### 5. 规则写得太死
比如只适用于一种日期格式，却拿去匹配所有日期。

### 6. 没先看原始数据
规则抽取最怕“闭眼写规则”。

教学里最重要的一条是：
- 先看数据长什么样
- 再写规则
- 再检查规则漏掉了什么、误抓了什么

## 十一、规则抽取和后面的工作台有什么关系？
### 和机器学习的关系
规则法是“人工告诉计算机怎么找”。

机器学习是“给计算机大量样本，让它自己学规律”。

在简单、稳定的场景里，规则法往往更快。

### 和 spaCy 的关系
后面的 `spaCy工作台` 会让你看到：
- 机器学习模型给出词性、实体、句法
- 然后规则再利用这些结果去做更细的提取

也就是说：
- 底层模型负责标注
- 上层规则负责筛选

### 和信息抽取的关系
规则抽取本身就是信息抽取的一种基础形式。

你现在学的是最简单、最直观的一种信息抽取方式。
