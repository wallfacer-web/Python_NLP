# 类

## 这页讲什么

这页讲 Python 里的类。  
对初学者来说，类不是最先必须掌握的内容，但它能帮助你理解：怎样把“一个对象的特征和动作”放在一起描述。

## 一、什么是类

如果说函数像一个小助手，那么类更像一张“设计图纸”。

设计图纸本身不是实物，但它规定了：
- 这一类东西应该有哪些特征
- 它能做什么动作

例如，“狗”这个类可以规定：
- 名字
- 年龄
- 能坐下
- 能叫

## 二、什么是对象

对象就是根据“类”这张图纸造出来的具体实例。

例如：
- `Dog` 是类
- 你家的“小白”是一只具体的狗，它就是对象

也可以这样理解：
- 类是模板
- 对象是按模板做出来的具体东西

## 三、什么是属性

属性是对象的特征。

比如一只狗有：
- 名字
- 年龄

这些就是属性。

## 四、什么是方法

方法是对象能执行的动作。  
方法本质上就是写在类里面的函数。

比如：
- 狗可以坐下
- 狗可以打滚

这些就是方法。

## 五、最简单的类例子

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def sit(self):
        print(self.name + " 正在坐下。")
```

这里先不用怕复杂，先抓住四个点：

- `class Dog:` 表示定义一个类
- `__init__` 是创建对象时自动运行的部分
- `self.name`、`self.age` 是属性
- `sit()` 是方法

## 六、如何创建对象

```python
my_dog = Dog("小白", 3)
print(my_dog.name)
my_dog.sit()
```

这里：
- `my_dog` 是一个对象
- 它的名字是 `"小白"`
- 它的年龄是 `3`
- 它可以执行 `sit()` 这个动作

## 七、为什么要学类

函数更像“处理事情的方法”。  
类更像“描述一类对象的结构”。

当你需要管理很多相关数据和动作时，类会比较有用。

例如以后你想描述：
- 学生
- 书籍
- 餐馆
- 文本对象

类会让这些信息更有条理。

## 八、类和函数的区别

一句话总结：

- 函数是一段处理数据的独立工序
- 类是数据和相关工序的结合体

更直白地说：
- 函数更像工具
- 类更像“带属性和动作的对象模板”

## 九、最适合初学者的类练习

### 练习1：创建一个餐馆类

要求：
- 类名：`Restaurant`
- 属性：`restaurant_name`、`cuisine_type`
- 方法：`open_restaurant()`

参考：

```python
class Restaurant:
    def __init__(self, restaurant_name, cuisine_type):
        self.restaurant_name = restaurant_name
        self.cuisine_type = cuisine_type

    def open_restaurant(self):
        print("餐馆正在营业。")

my_restaurant = Restaurant("小城面馆", "面食")
print(my_restaurant.restaurant_name)
my_restaurant.open_restaurant()
```

### 练习2：创建一个学生类

要求：
- 类名：`Student`
- 属性：`name`、`major`
- 方法：`introduce()`

参考：

```python
class Student:
    def __init__(self, name, major):
        self.name = name
        self.major = major

    def introduce(self):
        print("大家好，我叫" + self.name + "，我的专业是" + self.major)

student_1 = Student("张丽", "汉语言文学")
student_1.introduce()
```

## 十、初学者常见错误

- 把类和对象混为一谈
- 忘记写 `self`
- 创建对象时参数数量不对
- 方法定义了，但不会调用
- 一开始就想把类写得很复杂

## 十一、这一页学完后你应会什么

- 知道类像设计图纸
- 知道对象是类造出来的具体实例
- 知道属性是特征
- 知道方法是动作
- 能照着例子写最简单的类
