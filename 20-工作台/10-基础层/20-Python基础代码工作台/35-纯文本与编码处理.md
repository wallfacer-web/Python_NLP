# 纯文本与编码处理

## 这页讲什么

这页讲：
- 什么是纯文本
- 为什么自然语言处理必须先会处理文本文件
- 什么是 `utf-8`
- 什么是 `gbk`
- 为什么会乱码
- 如何读取、逐行读取、写入和追加文本文件

## 一、什么是纯文本

纯文本就是只包含“文字内容”的文件，不包含字体、字号、颜色、排版这些复杂格式。

常见的纯文本文件有：
- `.txt`
- `.py`
- 某些 `.csv`
- 某些 `.json`

对于初学者来说，可以先这样理解：

- Word 文档更像“排版好的成品”
- 纯文本更像“只有文字本体的原稿”

而自然语言处理最常处理的，就是这种“原稿式文本”。

## 二、为什么文本文件处理是 NLP 的基础

自然语言处理并不是一上来就做情感分析、分类、问答。

真正的第一步通常是：

1. 找到文本文件
2. 成功打开它
3. 成功读出内容
4. 确认没有乱码
5. 把内容放进 Python 字符串

只有这一步成功，后面的：
- 分词
- 统计
- 清洗
- 分类
- 摘要

才有可能发生。

## 三、什么是编码

计算机并不直接认识“汉字”和“英文单词”。  
它真正处理的是数字和字节。

编码可以理解成：

- 一套把“文字”翻译成“计算机数字”的规则

如果保存文件时和读取文件时使用的规则不一致，就容易出现：
- 乱码
- 报错

## 四、UTF-8 和 GBK 的区别

## 1. UTF-8

`utf-8` 是现在最通用、最推荐的编码方式。

优点：
- 跨平台
- 支持中文和英文
- 现代 Python 项目几乎都优先使用它

所以课程里默认优先使用：

```python
encoding="utf-8"
```

## 2. GBK

`gbk` 是 Windows 中文环境中很常见的一种编码方式。

它常见于：
- 老 txt 文件
- 老教材附带文本
- 某些中文 Windows 环境导出的文件

## 3. 最实用的规则

对初学者最重要的不是记概念，而是记规则：

- 优先用 `utf-8`
- 如果中文旧文件打不开，再试 `gbk`

## 五、为什么会乱码

乱码的本质是：

- 文件保存时用了一种编码
- 读取时却用了另一种编码

例如：
- 文件本来是 `gbk`
- 你却按 `utf-8` 去读

这时 Python 可能：
- 读出乱码
- 或直接报 `UnicodeDecodeError`

你可以把它理解成：
- 文件写的时候用的是一种翻译字典
- 读取时却拿了另一种字典去解读

当然就会出问题。

## 六、最推荐的文件打开方式

推荐初学者固定使用这种写法：

```python
with open("文件名.txt", "r", encoding="utf-8") as f:
    text = f.read()
```

这里：
- `open()`：打开文件
- `"r"`：读取模式
- `encoding="utf-8"`：指定编码
- `with ... as ...`：自动帮你关文件

## 七、四种最常用操作

## 1. 读取整个文件

```python
with open("diary.txt", "r", encoding="utf-8") as f:
    content = f.read()

print(content)
```

## 2. 逐行读取文件

```python
with open("diary.txt", "r", encoding="utf-8") as f:
    for line in f:
        print(line.strip())
```

这种方式特别适合：
- 日记
- 评论
- 按行存储的语料

## 3. 写入文件

```python
with open("diary.txt", "w", encoding="utf-8") as f:
    f.write("这是我的第一篇 Python 日记。")
```

注意：
- `"w"` 会覆盖原文件内容
- 如果文件不存在，会自动创建

## 4. 追加文件

```python
with open("diary.txt", "a", encoding="utf-8") as f:
    f.write("\n补充：今天我学会了读取文本文件。")
```

这里：
- `"a"` 表示追加
- 不会删除原有内容

## 八、最适合初学者的例子

### 例子1：写下第一篇日记

```python
with open("diary.txt", "w", encoding="utf-8") as f:
    f.write("这是我的第一篇 Python 日记！\n")
    f.write("今天开始学习自然语言处理的基础。")

print("日记写入成功！")
```

### 例子2：读出刚才的日记

```python
with open("diary.txt", "r", encoding="utf-8") as f:
    content = f.read()
    print("日记的内容是：")
    print(content)
```

### 例子3：给日记补充内容

```python
with open("diary.txt", "a", encoding="utf-8") as f:
    f.write("\n补充：我发现 Python 读写文件非常简单。")

print("新内容追加完毕！")
```

### 例子4：逐行审查文本

```python
print("开始逐行读取：")

with open("diary.txt", "r", encoding="utf-8") as f:
    for line in f:
        clean_line = line.strip()
        print("读取到一行 ->", clean_line)
```

### 例子5：尝试读取旧中文文件

```python
with open("old_text.txt", "r", encoding="gbk") as f:
    content = f.read()

print(content)
```

## 九、适合课堂使用的练习

### 练习1：写一份三行自我介绍

要求：
- 新建 `intro.txt`
- 写入三行内容

### 练习2：把 `intro.txt` 完整读出来

要求：
- 使用 `.read()`

### 练习3：逐行读取 `intro.txt`

要求：
- 使用 `for line in f`
- 打印时去掉换行符

### 练习4：给自我介绍追加一句话

要求：
- 使用 `"a"` 模式

### 练习5：观察乱码现象

要求：
- 先用 `utf-8` 读取
- 再尝试用错误编码读取
- 记录现象

教学提示：
- 不一定要求所有机器都真的制造乱码
- 但应让学生知道乱码来自“编码不匹配”

### 练习6：把一篇小文本复制到新文件

要求：
- 读取 `source.txt`
- 保存到 `copy.txt`

参考：

```python
with open("source.txt", "r", encoding="utf-8") as f:
    content = f.read()

with open("copy.txt", "w", encoding="utf-8") as f:
    f.write(content)
```

## 十、初学者常见错误

- 忘记写 `encoding="utf-8"`
- 文件不在当前目录
- 文件名拼错
- 用 `"w"` 时覆盖了旧内容
- 把追加模式和写入模式混淆
- 没理解为什么会乱码

## 十一、这一页学完后你应会什么

- 知道什么是纯文本
- 知道为什么 NLP 先要会读文本
- 知道 `utf-8` 和 `gbk` 的基本区别
- 知道乱码来自编码不匹配
- 能读取、逐行读取、写入、追加文本文件
