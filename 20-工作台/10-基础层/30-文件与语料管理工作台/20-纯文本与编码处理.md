# 纯文本与编码处理

## 这页讲什么

这页讲：
- 什么是纯文本
- 为什么 NLP 常常先处理 `.txt`
- 什么是编码
- `utf-8` 和 `gbk` 的区别
- 为什么会乱码
- 如何读取、逐行读取、写入、追加文本文件

## 一、什么是纯文本

纯文本就是只包含文本字符的文件，不带字体、字号、颜色、排版等复杂格式。

常见的纯文本文件有：
- `.txt`
- `.py`
- 某些日志文件
- 某些导出的语料文本

可以先这样理解：
- Word 文档更像排版好的成品
- 纯文本更像只保留文字内容的原稿

自然语言处理最常处理的，往往就是这种“原稿式文本”。

## 二、为什么 NLP 首先处理的常常是 txt 文本

因为计算机做 NLP 时，最先需要的是“干净可读的文字内容”，而不是复杂排版。

像 Word、网页、PDF 往往带有：
- 样式
- 标签
- 页面结构
- 多余格式信息

而 `.txt` 更接近语言本身。  
所以在很多任务里，第一步就是：

1. 把各种格式转换成纯文本
2. 把纯文本读进 Python
3. 再做清洗、统计、分析

## 三、什么是编码

计算机底层并不直接认识汉字和英文，它只认识字节。

编码可以理解成：
- 把人类文字翻译成计算机能存储的数字规则

如果保存文件和读取文件时用的不是同一套规则，就容易出现：
- 乱码
- 报错

## 四、UTF-8 和 GBK 的区别

### 1. UTF-8

`utf-8` 是现代最常用、最推荐的编码。

优点：
- 跨平台
- 支持中英文
- 兼容性好
- 几乎所有现代 Python 项目都优先用它

课程里默认优先使用：

```python
encoding="utf-8"
```

### 2. GBK

`gbk` 是 Windows 中文环境里常见的一种编码方式。

它常出现在：
- 老 txt 文件
- 老教材附带资料
- 早期中文 Windows 导出的文本

### 3. 最实用的规则

对于初学者，最重要的是记住：

- 先试 `utf-8`
- 如果中文旧文件打不开，再试 `gbk`

## 五、为什么会乱码

乱码的本质是：

- 文件是按一种编码保存的
- 你却用另一种编码去读取

这就像：
- 写的时候用的是一本字典
- 读的时候却拿了另一本字典去查

当然就会出问题。

轻则乱码，重则报：

```text
UnicodeDecodeError
```

## 六、推荐的文件操作写法

初学者建议固定使用：

```python
with open("文件名.txt", "r", encoding="utf-8") as f:
    text = f.read()
```

这里：
- `open()`：打开文件
- `"r"`：读取
- `"w"`：写入
- `"a"`：追加
- `encoding="utf-8"`：指定编码
- `with ... as ...`：自动关闭文件

## 七、最常用的四种操作

### 1. 读取整个文件

```python
with open("hello.txt", "r", encoding="utf-8") as f:
    content = f.read()

print(content)
```

### 2. 逐行读取

```python
with open("hello.txt", "r", encoding="utf-8") as f:
    for line in f:
        print(line.strip())
```

这特别适合：
- 日记
- 按行保存的评论
- 小型语料

### 3. 写入文件

```python
with open("hello.txt", "w", encoding="utf-8") as f:
    f.write("这是我写给世界的第一行代码。\n")
```

注意：
- `"w"` 会覆盖原有内容

### 4. 追加文件

```python
with open("hello.txt", "a", encoding="utf-8") as f:
    f.write("这是我追加的第二行代码。\n")
```

注意：
- `"a"` 不会删掉原来的内容

## 八、适合课堂演示的例子

### 例子1：写入纯文本文件

```python
with open("diary.txt", "w", encoding="utf-8") as f:
    f.write("这是我的第一篇 Python 日记。\n")
    f.write("今天开始学习自然语言处理的基础。")
```

### 例子2：读取整个文件

```python
with open("diary.txt", "r", encoding="utf-8") as f:
    content = f.read()
    print(content)
```

### 例子3：追加新内容

```python
with open("diary.txt", "a", encoding="utf-8") as f:
    f.write("\n补充：我发现 Python 读写文件非常简单。")
```

### 例子4：逐行读取

```python
with open("diary.txt", "r", encoding="utf-8") as f:
    for line in f:
        print("读取到一行：", line.strip())
```

### 例子5：用 pathlib 读取文件

```python
from pathlib import Path

file_path = Path("diary.txt")
content = file_path.read_text(encoding="utf-8")
print(content)
```

### 例子6：列出当前目录下所有 txt 文件

```python
import os

for file_name in os.listdir("."):
    if file_name.endswith(".txt"):
        print(file_name)
```

### 例子7：尝试用 gbk 读取旧文件

```python
with open("old_text.txt", "r", encoding="gbk") as f:
    content = f.read()
    print(content)
```

### 例子8：复制文本内容到新文件

```python
with open("source.txt", "r", encoding="utf-8") as f:
    content = f.read()

with open("copy.txt", "w", encoding="utf-8") as f:
    f.write(content)
```

## 十、常见错误

- 忘记写 `encoding="utf-8"`
- 文件路径不对
- 文件名拼错
- 用 `"w"` 时不小心覆盖旧内容
- 逐行读取时不理解换行符
- 不知道为什么会乱码
