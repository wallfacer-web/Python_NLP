# 语料命名与目录规范

## 这页讲什么？
这页讲：
- 为什么文件命名和目录结构会影响 NLP 工作
- 原始数据和处理后数据为什么必须分开
- 初学者最适合采用什么样的项目目录
- 文件命名时应该遵守哪些规则
- 初学者最常犯哪些错误

## 一、为什么命名和目录规范这么重要？
可以把你的语料项目想成一间图书室。

如果书都乱堆在地上，封面有的用中文、有的用空格、有的只有“新文件(3)”，那你很快就会遇到这些问题：
- 自己都找不到某一份文本
- 代码遍历目录时把不该读的文件也读进来了
- 处理后的文件覆盖了原始文件
- 过几周回头看，已经不知道哪个版本是干净语料

在 NLP 项目里，目录混乱不是“小问题”，而是后面很多错误的根源。

## 二、原始数据和处理后数据为什么必须分层？
这是最重要的一条规则：

不要直接改原始文件。

原因很简单：
- 原始数据是你的“原件”
- 清洗后的数据是“工作副本”
- 结果文件是“输出物”

如果不分层，很容易出现：
- 把原始文本洗坏了
- 不知道哪些文件已经处理过
- 结果图表和源数据混在一起

## 三、一个适合初学者的目录结构
下面这个结构很适合课程项目：

```text
my_nlp_project/
├── 1_raw_data/
│   ├── news_001.txt
│   ├── news_002.txt
│   └── comments_001.csv
├── 2_clean_data/
│   ├── news_001_clean.txt
│   └── comments_clean.csv
├── 3_code/
│   ├── 01_read_files.py
│   ├── 02_clean_text.py
│   └── 03_word_count.py
└── 4_results/
    ├── top_words.csv
    └── wordcloud.png
```

这个结构的好处是：
- 一眼就能看出数据在哪、代码在哪、结果在哪
- 原始数据不容易被误改
- 每一步处理都能留下痕迹

## 四、文件命名的基本规则
### 1. 尽量使用小写
推荐：

```text
news_001.txt
student_comments.csv
```

不推荐：

```text
News_001.txt
Student Comments.csv
```

### 2. 用下划线，不要乱用空格
推荐：

```text
movie_reviews.txt
course_feedback_01.csv
```

不推荐：

```text
movie reviews.txt
course feedback 01.csv
```

### 3. 文件名要表达清楚内容
推荐：

```text
news_politics_001.txt
weibo_comments_march.csv
```

不推荐：

```text
新建文本文档.txt
data1.txt
test.txt
```

### 4. 最好带上编号或日期

```text
speech_001.txt
speech_002.txt
news_20260323_001.txt
```

## 五、课堂里最值得强调的三条规则
### 规则1：原始数据不直接修改
原始文件放到 `1_raw_data`，清洗后另存到 `2_clean_data`。

### 规则2：代码、数据、结果分开
不要把 `.py`、`.txt`、`.png` 都堆在一个文件夹里。

### 规则3：命名要让“未来的自己”也能看懂
你今天看得懂，不代表两周后还看得懂。

## 六、教师课堂演示案例
### 案例1：乱目录 vs 规范目录

```text
我的文本/
├── 新建文本文档.txt
├── test.py
├── 图片1.jpg
├── article final final.txt
└── 课程作业.docx
```

对比：

```text
mini_project/
├── 1_raw_data/
├── 2_clean_data/
├── 3_code/
└── 4_results/
```

### 案例2：给乱命名文件改名
把下面这些文件名改成规范形式：

```text
新建文本文档.txt
我的评论.txt
最终版.txt
final_final_use_this.txt
```

可以改成：

```text
comment_001.txt
comment_002.txt
comment_003.txt
comment_004.txt
```

## 七、适合初学者的 Python 小例子
### 例子1：列出一个目录中的文件

```python
from pathlib import Path

folder = Path("1_raw_data")

for file_path in folder.iterdir():
    print(file_path.name)
```

### 例子2：只找 txt 文件

```python
from pathlib import Path

folder = Path("1_raw_data")

for file_path in folder.glob("*.txt"):
    print(file_path.name)
```

### 例子3：检查目录是否存在

```python
from pathlib import Path

folder = Path("2_clean_data")

if folder.exists():
    print("文件夹存在")
else:
    print("文件夹不存在")
```

### 例子4：自动创建结果目录

```python
from pathlib import Path

result_folder = Path("4_results")
result_folder.mkdir(exist_ok=True)

print("结果目录已准备好")
```

## 九、常见错误
- 把原始文件和清洗后的文件放在一起
- 用 `data.txt`、`test.txt` 这类几乎没有信息量的名字
- 在文件名里混用空格、中文、特殊符号
- 代码遍历整个目录时，把图片和代码文件也读进去了
- 做了很多版本，却没有编号和日期

## 十、这页学完后你应该会什么？
- 知道为什么 NLP 项目必须做目录分层
- 会设计一个基础的语料项目结构
- 会给文本文件起清楚、可读、可管理的名字
- 会用最简单的 Python 代码检查目录和文件
