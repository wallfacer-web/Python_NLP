# 大模型文本处理工作台

## 工作台定位
前沿层。负责使用大语言模型处理文本任务，例如：
- 摘要

这一页不是为了让学生“只会问 AI”，而是让学生理解：
- 大模型为什么强

- 为什么仍然需要人工校验和结构化工作流

## 一、什么是大语言模型？
大语言模型可以理解成：

一个基于海量文本训练出来的语言处理系统。

它最直观的能力是：
- 能读文本
- 能生成文本
- 能根据指令完成很多不同类型的语言任务

对这门课来说，不必一开始陷进复杂数学，只要先理解：

它和前面传统 NLP 方法最大的差别是：
- 以前常常要为每个任务单独写规则或训练模型
- 现在很多任务可以直接通过自然语言指令完成

## 二、为什么它会改变文本处理方式？
因为它带来了一个很大的变化：

你不再总是需要为“摘要”“分类”“抽取”分别写三套完全不同的系统。

很多时候，你只需要：
- 把任务说明清楚
- 把文本给模型
- 让模型按要求输出

这就是为什么大模型会让文本处理从：
- 规则驱动
- 统计驱动

逐步转向：
- 指令驱动
- 生成驱动

## 三、典型任务有哪些？
### 1. 摘要
把长文压缩成短文。

### 2. 抽取
从文本中提取：
- 人名
- 时间
- 金额
- 事件
- 优缺点

### 3. 改写
把原文改成：
- 更正式
- 更简洁
- 更口语
- 指定风格

### 4. 问答
根据给定文本回答问题。

### 5. 分类
判断一段文本属于哪个类别。

所以从任务表面看，大模型几乎像是一个“通用文本处理器”。

## 四、什么是 Prompt？
`Prompt` 就是你给大模型的任务说明。

可以把它理解成：
- 你写给 AI 的工作单
- 你交给 AI 的任务指令

如果 Prompt 很模糊，模型输出通常也会很散。

如果 Prompt 很清楚，输出就更容易可控。

## 五、如何通俗理解 Prompt？
可以把大模型想成一个非常聪明、知识很多、但需要你把任务讲清楚的实习生。

例如你只说：

```text
总结一下
```

这条指令太空。

但如果你说：

```text
请把下面这段课程反馈总结成 3 个要点。
要求：
1. 每点不超过 20 字
2. 聚焦学生抱怨的问题
3. 用中文输出
```

这就清楚很多。

所以 Prompt 的本质是：
- 帮模型理解角色
- 帮模型理解任务
- 帮模型理解输出格式

## 六、为什么大模型看起来很强，但仍然会出错？
### 1. 会幻觉
大模型有时会编出：
- 不存在的事实
- 不存在的参考文献
- 原文里没有的信息

这就叫幻觉。

### 2. 有上下文限制
模型一次能处理的文本长度有限。

如果文本特别长，它可能：
- 漏看前文
- 忘记后文
- 只能处理其中一部分

### 3. 不是真正“理解世界”
它能生成很像正确答案的文本，但不代表它真的像人类一样理解事实。

### 4. 输出不稳定
同一个问题，多次问，结果可能不完全一样。

## 七、为什么人工校验非常重要？
因为大模型不是最终裁判，它更像：
- 强大的助手
- 副驾驶

不是：
- 最终责任人

所以在课程里必须强调：
- 模型给结果
- 人类做复核

尤其在这些场景里更不能直接信：
- 事实性摘要
- 结构化抽取
- 需要精确数据的分析

## 八、一个适合课堂的 Prompt 示例：分类
下面是一个最简单的伪代码思路：

```python
prompt = """
你是一个课程反馈分析助手。
请判断下面这段文本属于哪一类：
1. 教学内容
2. 作业安排
3. 设备问题

文本：
投影仪经常坏，影响上课体验。

只输出类别名称。
"""
```

这个 Prompt 的优点是：
- 任务清楚
- 类别明确
- 输出格式明确

## 九、一个适合课堂的 Prompt 示例：抽取

```python
prompt = """
请从下面文本中抽取：
1. 时间
2. 地点
3. 人物

文本：
2026年3月23日，李老师在上海举办了一场关于 Python NLP 的讲座。

请用 JSON 输出。
"""
```

这个例子适合讲：
- 大模型也能做抽取
- 但必须告诉它输出格式

## 十、一个适合课堂的 Prompt 示例：摘要

```python
prompt = """
请把下面文本总结成 3 个要点。
要求：
1. 每点不超过 15 个字
2. 只保留核心信息
3. 不要加入原文没有的内容
"""
```

这个例子很适合拿来和前面的“抽取式摘要”作比较。

## 十二、初学者最常见错误
### 1. 把大模型当搜索引擎
模型不一定知道最新事实，也不一定会给出真实来源。

### 2. Prompt 太空泛
没有说明：
- 任务目标
- 输出格式
- 角色
- 字数要求

### 3. 不检查结果真假
看它写得很流畅，就以为内容一定对。

### 4. 忽视格式约束
如果后续程序要读取 JSON，但你没要求“只输出 JSON”，模型可能会多说很多废话。

### 5. 以为大模型可以完全替代前面的工作台
这在课程里必须明确反对。

大模型很强，但它不是课程系统本体。

## 十三、它和前后工作台是什么关系？
### 和前面的关系
前面的 `文本清洗工作台` 仍然重要，因为脏文本会浪费 token，也会干扰输出质量。

前面的 `规则抽取工作台` 仍然重要，因为很多时候大模型输出后，还需要规则来校验和后处理。

前面的 `主题分析与摘要工作台` 仍然重要，因为传统方法能帮助学生理解：
- 摘要是什么
- 主题是什么
- 大模型到底替代了哪些步骤，又新增了哪些风险

### 和后面的关系
后面的 `RAG工作台` 会进一步解决大模型的两个核心问题：
- 知识不够新
- 容易幻觉

可以把 RAG 理解成：
- 给大模型加上检索和参考资料

所以这一页更像是：
- 大模型基础使用页
- RAG 的前置页
