# 异常处理

## 一句话定义
异常处理是程序遇到错误时的保护机制，用来避免一次报错就让整个流程中断。

## 通俗理解
可以把异常处理理解成程序的安全气囊。

如果某个文件坏了、路径错了、编码出错了，程序不一定要整个崩掉，可以先记录错误，再继续处理别的内容。

## 最小例子

```python
try:
    text = open("missing.txt", "r", encoding="utf-8").read()
except Exception as e:
    print("读取失败：", e)
```

## 常见误区
- 一报错就慌，不知道程序可以主动处理错误
- 什么都 `except`，但不去看具体原因
- 以为异常处理是高级技巧，其实批量读文件时非常实用
