# 虚拟环境

## 一句话定义
虚拟环境是为某个项目单独准备的一套 Python 运行环境，用来隔离不同项目的依赖版本。

## 通俗理解
可以把虚拟环境理解成每个项目自己的实验室。

这样一个项目装了 `spacy` 某版本，不会和另一个项目的依赖打架。

## 最小例子

```bash
python -m venv .venv
```

或者课程里也可以用：

```bash
conda create -n python_nlp python=3.11
```

## 常见误区
- 所有项目共用一个环境
- 明明装了库，但装在了别的环境里
- 不理解为什么“同一台电脑上会有多个 Python”
