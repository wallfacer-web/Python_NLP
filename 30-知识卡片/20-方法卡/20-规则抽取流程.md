# 规则抽取流程

## 方法定位
这张方法卡把规则抽取任务压缩成一个可操作流程，适合入门信息抽取任务。

## 适用场景
- 抽取日期、邮箱、网址、课程代码
- 从固定格式文本中提取字段
- 做小规模、可解释的抽取任务

## 不适用场景
- 文本表达非常自由、差异很大
- 需要深层语义理解的复杂抽取
- 规则难以稳定覆盖的大规模开放场景

## 四步流程

### 第一步：先定义要抽什么
不要一开始就写正则，先明确目标字段：
- 日期
- 邮箱
- 网址
- 课程名称
- 编号

这一步的关键是把“任务对象”写清楚。

### 第二步：先看样本，再找规律
抽 5 到 10 条真实文本，观察它们的共同模式。

例如日期可能有这些形式：
- `2026-03-23`
- `2026/03/23`
- `2026年3月23日`

规则不是凭空写出来的，而是从样本里总结出来的。

### 第三步：写最小可用规则
先写能跑通的基础规则，再慢慢扩展，不要一开始就追求覆盖所有情况。

最小例子：

```python
import re

text = "课程开始日期是2026-03-23，请及时提交。"
dates = re.findall(r"\d{4}-\d{2}-\d{2}", text)
print(dates)
```

### 第四步：整理成结构化结果
抽取不是把结果打印出来就结束，最好整理成列表、字典或表格。

最小例子：

```python
result = {
    "date": dates[0] if dates else None
}
print(result)
```

## 常见错误
- 不先看样本就直接写规则
- 想一次覆盖所有情况，导致规则过长、难维护
- 只看成功案例，不看失败案例
- 抽取后不做结构化整理
