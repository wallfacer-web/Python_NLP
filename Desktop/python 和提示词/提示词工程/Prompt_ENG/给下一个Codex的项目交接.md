课程的组织原则不是“按教材章节”，而是“按任务流程”。

## 3. 课程内容编辑原则
这个仓库不是“知识点堆砌”，而是“可运行的课程系统”。

后续编辑时请保持以下原则：

- 先结构，后内容
- 先工作流，后扩展页
- 页面尽量回答三个问题：
  - 它属于哪个层级
  - 它服务哪个工作台或模板
  - 它对应什么输出动作
- 概念卡不要只写定义，要有：
  - 一句话定义
  - 通俗理解
  - 最小例子
  - 常见误区
  - 服务工作台
  - 对应输出动作
- 面向对象是零基础、文科背景学生，表达必须慢、清楚、可运行

## 4. NotebookLM 怎么用
仓库里已经有一份专门说明：

- [[给新Codex的NotebookLM使用说明]]

真正要记住的是：

### 4.1 认证方式
不要把这个技能理解成“手动登录网页步骤”，它的核心是 CLI 登录。

标准流程：

```bash
notebooklm login
```

然后：
1. 浏览器里完成 Google 登录
2. 真正进入 NotebookLM 首页
3. 回到终端按一次回车
4. 认证保存到本地 `storage_state.json`

### 4.2 认证文件
默认位置：

```text
~/.notebooklm/storage_state.json
```

### 4.3 使用顺序
当用户要求用 NotebookLM 时，默认按下面顺序执行：

```bash
notebooklm status
notebooklm list
notebooklm ask "你的问题"
```

### 4.4 当前最常见问题
`notebooklm status` 有时能成功，但 `notebooklm ask` 仍可能提示：

```text
Authentication expired or invalid
```

这时不要误判为“技能坏了”，通常是登录态过期了，要重新执行：

```bash
notebooklm login
```

### 4.5 给 NotebookLM 的提示词要优化
这是一个已经验证过的经验，必须继续执行。

不要只问：
- “介绍一下变量”
- “讲讲 NLTK”

而要问得具体：
- 面向什么学生
- 讲哪些部分
- 至少给多少例子
- 是否要包含常见错误
- 是否要强调教学语言

好的提示词结构通常是：
- 对象：零基础、文科背景、正在学 Python 与 NLP 的学生
- 任务：详细解释某个概念
- 输出要求：分点说明概念、作用、用法、关系、误区
- 例子要求：至少 8 到 12 个短小可运行例子
- 风格要求：教学语言，多例子，少抽象

## 6. 已经踩过并修掉的网页问题
之前最重要的坑已经修过，下一个 Codex 不要重复排查旧问题。

### 6.1 工作台 404
原因不是页面没写，而是 Quartz 原来输出为单个 `.html` 文件，但站内链接使用无扩展名路径，GitHub Pages 对中文路径没有兜底。

已修：
- 普通内容页改成目录式输出
- 输出形态改成：

```text
页面路径/index.html
```

这样无扩展名内部链接能稳定工作。

### 6.2 GitHub Actions 构建失败
Linux runner 上曾出现 Quartz CLI 权限问题。

已修：
- 构建命令改成显式：

```bash
node ./quartz/bootstrap-cli.mjs build
```

### 6.3 知识卡片导航 404
原因是上层 MOC 页里的双链路径写法会被 Quartz 解析偏。

已修：
- `30-知识卡片/知识卡片MOC.md`
- `30-知识卡片/10-概念卡/00-概念卡MOC.md`

### 6.4 全站链接检查
之前已经做过批量检查，公开站链接在当时测试结果里是通的。

但如果以后又出现 404，请优先排查：
- 新建页面的 wikilink 是否写成了模糊短链
- 页面标题和文件名是否不一致
- MOC 页是否写了会被 Quartz 误解析的相对链接
