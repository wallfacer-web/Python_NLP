# 作业说明

## 作业的作用

这门课的作业不是为了重复课堂定义，而是为了让学生把工作台中的方法真正跑通。
每一次作业都应尽量对应一个明确工作台、一个模板和一个输出动作。

## 作业类型

### 1. 基础练习作业

适用阶段：
- Python 基础代码
- 文件与语料管理
- Pandas 文本数据处理

典型输出：
- Python 脚本
- 截图或命令结果
- 简短运行说明

### 2. 方法实验作业

适用阶段：
- 文本清洗
- 规则抽取
- 词频统计
- spaCy

典型输出：
- 处理前后对比
- 提取结果
- 图表或表格
- 方法说明

### 3. 应用任务作业

适用阶段：
- 文本分类
- 情感分析
- 主题分析与摘要
- ASR 与 TTS

典型输出：
- 实验结果
- 评价指标
- 错误分析
- 简短结论

### 4. 前沿项目作业

适用阶段：
- 大模型文本处理
- RAG

典型输出：
- Prompt 设计
- 检索结果
- 最终回答
- 失败案例与反思

## 提交规则

- 每次作业必须标明姓名、学号、工作台名称
- 代码、数据、结果尽量分文件夹保存
- 文件命名尽量统一，不要使用随意缩写
- 如果使用模板，提交内容应按模板结构组织

## 建议提交结构

```text
作业名称/
├─ code/
├─ data/
├─ output/
└─ report.md
```

## 教师批改重点

- 能不能运行
- 输出是否对应任务要求
- 是否解释清楚方法步骤
- 是否能说明结果和问题

## 学生最常见问题

- 只交代码，不交结果说明
- 只交结果，不交方法过程
- 文件命名混乱
- 代码能跑，但和作业要求不对应
